ModbusManager User Manual

Complete guide for installing, configuring and using ModbusManager — a professional Modbus Poll tool for Windows.

Introduction

ModbusManager is a desktop application for testing, debugging and monitoring Modbus RTU and TCP devices. It runs on Windows 10/11 and provides a clean, modern interface for industrial engineers and automation professionals.

Key Features

  • Modbus TCP — connect to Modbus devices over Ethernet/IP
  • Modbus RTU — connect over serial RS-232/RS-485
  • Modbus ASCII — connect using ASCII framing over serial
  • Multiple Poll Windows — monitor several register ranges simultaneously
  • Real-time TX/RX Log — see every byte sent and received
  • Chart View — visualize register values over time
  • Saved Profiles — save and reload device configurations instantly
  • CSV Export — export log data for further analysis
  • 100% offline — no internet connection required during operation

Installation

1
Download the installer

Download Modbus.Manager.Setup.1.0.0.exe from modbusmanager.com.

2
Run the installer

Double-click the downloaded file. If Windows SmartScreen appears, click "More info" → "Run anyway".

3
Choose installation folder

The default folder is C:\Program Files\ModbusManager. You can change this if needed.

4
Complete installation

Click Install and wait for the process to finish. A desktop shortcut will be created automatically.

5
Launch ModbusManager

Double-click the desktop shortcut or find ModbusManager in the Start menu.

Note: Windows Firewall may ask permission to allow ModbusManager through the network. Click Allow to enable Modbus TCP connections.

License & Trial

ModbusManager includes a 14-day free trial with full functionality. After the trial expires, you need to activate a license to continue using the software.

Activating your license

1
Open the License window

Click the License button in the top-right corner of the application.

2
Copy your Machine ID

Your unique Machine ID is shown in the License window. Click Copy to copy it to clipboard.

3
Send your Machine ID

Email your Machine ID to support@modbusmanager.com. You will receive your license key automatically within minutes.

4
Enter your license key

Type or paste your license key (format: MBUS-XXXX-XXXX-XXXX) into the input field and click Activate.

Important: The license key is tied to your specific machine. Do not share it with others — it will not work on a different computer.

Connecting via Modbus TCP

Modbus TCP connects to devices over Ethernet using the standard TCP/IP protocol. This is the most common connection type for modern PLCs, inverters and energy meters.

Connection parameters

ParameterDescriptionDefault
HostIP address of the Modbus device192.168.1.100
PortTCP port number502
Slave IDModbus unit identifier (1–247)1

Steps

1
Select TCP mode

In the connection bar at the top, select FC03 Holding Reg or your desired function code.

2
Enter the IP address and port

Type the device IP address and port. Default Modbus port is 502.

3
Set Slave ID

Enter the Slave ID of your device. If unsure, try 1.

4
Click Connect

The status bar at the bottom will show Connected (TCP) if successful.

Connecting via Modbus RTU

Modbus RTU connects over a serial port (RS-232 or RS-485). This is used with older PLCs, drives and meters that do not have Ethernet connectivity.

Connection parameters

ParameterDescriptionTypical value
Serial PortCOM port of your USB-to-serial adapterCOM3, COM4
Baud RateCommunication speed in bits/second9600 or 19200
Data BitsNumber of data bits per frame8
ParityError checking methodNone
Stop BitsNumber of stop bits1
Slave IDAddress of the target device1
Tip: Check your device manual for the correct baud rate and parity settings. The most common setting is 9600 baud, 8 data bits, no parity, 1 stop bit (9600-8-N-1).

Finding your COM port

Open Device Manager (Win + X → Device Manager) → expand Ports (COM & LPT) → note the COM port number of your USB-serial adapter.

Connecting via Modbus ASCII

Modbus ASCII uses human-readable ASCII characters instead of binary encoding. It is slower than RTU but easier to debug manually. Typically used with older equipment.

The connection parameters are the same as RTU, but the typical Data Bits setting is 7. Select ASCII mode in the connection settings.

Poll Windows

Poll Windows allow you to monitor multiple register ranges at the same time. Each window can have its own function code, address range and polling interval.

Adding a Poll Window

1
Click "+ Window"

In the top toolbar, click the + Window button to create a new poll window.

2
Configure the window

Set the Function Code (FC01–FC04), starting Address, Quantity of registers and Slave ID.

3
Click Start

Click the Start button to begin automatic polling at the configured interval.

Poll Window columns

Function Codes

Modbus uses function codes (FC) to specify which type of data to read or write.

CodeNameDescriptionRead/Write
FC01 Read Coils Digital output status (ON/OFF) Read
FC02 Read Discrete Inputs Digital input status (ON/OFF) Read only
FC03 Read Holding Registers 16-bit read/write registers (most common) Read/Write
FC04 Read Input Registers 16-bit read-only registers (sensors, measurements) Read only
FC05 Write Single Coil Set a single digital output ON or OFF Write
FC06 Write Single Register Write a value to a single holding register Write
FC15 Write Multiple Coils Set multiple coils at once Write
FC16 Write Multiple Registers Write values to multiple holding registers at once Write

Writing Registers & Coils

ModbusManager allows you to write values to Modbus devices using the Write tab in any Poll Window.

Writing a single register (FC06)

  1. Open a Poll Window with FC03 (Holding Registers)
  2. Click the Write tab
  3. Enter the register address and the value to write
  4. Click Write
Warning: Writing incorrect values to a device can cause unexpected behaviour or damage. Always consult your device manual before writing to registers.

Chart View

The Chart tab in each Poll Window shows register values plotted over time. This is useful for monitoring analog signals, temperatures, pressures and other continuously changing values.

  • Click the Chart tab in any Poll Window to see the live chart
  • The chart automatically scales to the current data range
  • Up to the last 100 values are displayed

TX/RX Log

The TX/RX Log at the bottom of the screen shows every Modbus frame sent and received in real time. This is essential for debugging communication issues.

Log entry format

[14:06:43] → TX [FC03] Addr:5 Qty:10 (slave 50)
[14:06:43] ← RX [FC03] 10 regs OK (55ms)
SymbolMeaning
→ TXData sent from ModbusManager to the device
← RXData received from the device
[FC03]Function code used
Addr:5Starting register address
Qty:10Number of registers requested
(55ms)Response latency

Status bar

The bottom status bar shows cumulative statistics: TX (frames sent), RX (frames received), Errors and average Latency.

Profiles

Profiles allow you to save your entire session configuration — all Poll Windows, connection settings and register descriptions — and reload them instantly.

Saving a profile

  1. Set up your connection and Poll Windows as desired
  2. Click the save icon (💾) next to a Poll Window name
  3. Enter a name for the profile
  4. The profile is saved automatically

Loading a profile

  1. Click the load icon next to a Poll Window
  2. Select a saved profile from the list
  3. The window configuration is restored instantly
Tip: Profiles are stored locally on your machine in the profiles/ folder next to the application. You can copy them to share with colleagues.

CSV Export

You can export the TX/RX log to a CSV file for further analysis in Excel or other tools.

  1. Click the Save log button in the TX/RX Log panel
  2. Choose a filename and location
  3. The log is saved as a plain text file in the logs/ folder

Frequently Asked Questions

Why does Windows show a security warning when installing?

Windows SmartScreen warns about installers that are not yet widely distributed. Click "More info" → "Run anyway" to proceed. ModbusManager is safe to install.

My device is not responding — what should I check?

  • Verify the IP address (TCP) or COM port (RTU) is correct
  • Check that the Slave ID matches your device configuration
  • For RTU: confirm the baud rate, parity and stop bits match the device
  • For TCP: check that port 502 is not blocked by a firewall
  • Try reducing the number of registers in a single poll (Quantity)

Can I use ModbusManager on a network without internet?

Yes. ModbusManager works completely offline. Internet is only required to download the installer and activate the license once.

How do I find my Machine ID?

Open ModbusManager → click the License button in the top right corner → your Machine ID is displayed. Click Copy to copy it to clipboard.

Can I use one license on multiple computers?

Each license is tied to one machine. If you need multiple seats, contact us at support@modbusmanager.com for team pricing.

Where are profiles stored?

Profiles are stored in the profiles/ folder in the ModbusManager installation directory. You can back them up or copy them to other machines.
